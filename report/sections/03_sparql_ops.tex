\chapter{Opera\c{c}\~oes sobre os dados (SPARQL)}

\section{Vis\~ao geral da organiza\c{c}\~ao}

As opera\c{c}\~oes SPARQL do projeto foram reorganizadas por dom\'inio funcional na pasta \texttt{webapp/baseball/sparql\_queries/}. Esta divis\~ao permite separar claramente as queries de jogadores, equipas, ligas, grafos, vistas auxiliares e quiz.

\begin{itemize}[leftmargin=*]
  \item \texttt{players.py}: cat\'alogo de jogadores, filtros, detalhe biogr\'afico, hist\'orico desportivo, pr\'emios, All-Star Games, Hall of Fame e sal\'arios.
  \item \texttt{teams.py}: diret\'orio de franquias, hist\'orico por \'{e}poca, postseason, plant\'eis, managers, All-Stars e pr\'emios por equipa.
  \item \texttt{leagues.py}: navega\c{c}\~ao por liga, equipas da liga, s\'eries e l\'ideres estat\'isticos.
  \item \texttt{graphs.py}: queries usadas para construir os grafos relacionais dos detalhes de jogador e equipa.
  \item \texttt{misc.py}: listas auxiliares e rankings globais, como sal\'arios, Hall of Fame, managers e l\'ideres agregados.
  \item \texttt{quiz.py}: banco de perguntas do quiz e verifica\c{c}\~ao de respostas com \texttt{SELECT} e \texttt{ASK}.
  \item \texttt{edit\_service.py}: opera\c{c}\~oes \texttt{SPARQL UPDATE} para aprovar edi\c{c}\~oes sobre jogadores e equipas.
\end{itemize}

O ficheiro \texttt{webapp/baseball/sparql.py} continua apenas como superf\'icie de compatibilidade, reexportando as fun\c{c}\~oes dos m\'odulos internos.

\section{Queries por p\'agina}

Nesta sec\c{c}\~ao, as opera\c{c}\~oes s\~ao apresentadas por p\'agina ou contexto funcional da aplica\c{c}\~ao: rota ou contexto de utiliza\c{c}\~ao, fun\c{c}\~ao Python associada, descri\c{c}\~ao e tipo de opera\c{c}\~ao SPARQL. A listagem cobre as queries SPARQL de alto n\'ivel efetivamente usadas pelas p\'aginas e fluxos da aplica\c{c}\~ao. Fun\c{c}\~oes auxiliares que apenas agregam resultados em Python, como \texttt{get\_player\_batting\_summary()} e \texttt{get\_player\_pitching\_summary()}, n\~ao aparecem como entradas independentes porque n\~ao executam uma query adicional.

\subsection{Jogadores}

\begin{itemize}[leftmargin=*]
  \item \textbf{/players/ -- Lista paginada de jogadores}
  
  Fun\c{c}\~ao: \texttt{get\_players\_catalog()}
  
  Descri\c{c}\~ao: devolve o subconjunto paginado do cat\'alogo com os campos usados nos cartões da interface.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?playerID ?bbrefID ?name ?nameFirst ?nameLast ?nameGiven
       ?birthCountry ?bats ?throws ?debut ?finalGame
WHERE {
    ?player a bb:Player ;
            bb:playerID ?playerID ;
            foaf:name ?name .
    OPTIONAL { ?player bb:nameFirst ?nameFirst . }
    OPTIONAL { ?player bb:nameLast ?nameLast . }
    OPTIONAL { ?player bb:nameGiven ?nameGiven . }
    OPTIONAL { ?player bb:bbrefID ?bbrefID . }
    OPTIONAL { ?player bb:birthCountry ?birthCountry . }
    OPTIONAL { ?player bb:bats ?bats . }
    OPTIONAL { ?player bb:throws ?throws . }
    OPTIONAL { ?player bb:debut ?debut . }
    OPTIONAL { ?player bb:finalGame ?finalGame . }
}
ORDER BY ?name ?playerID
\end{Verbatim}

  \item \textbf{/teams/ -- Diret\'orio de franquias}
  
  Fun\c{c}\~ao: \texttt{get\_team\_franchise\_catalog()}
  
  Descri\c{c}\~ao: constr\'oi a grelha principal de equipas/franquias com informa\c{c}\~ao da \'{u}ltima \'{e}poca conhecida.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>

SELECT ?franchiseId ?franchiseName ?active
       ?year ?teamName ?teamId ?teamIdBr ?league ?division ?rank
       ?park ?attendance ?wins ?losses
WHERE {
    ?franchise a bb:Franchise ;
               bb:franchID ?franchiseId .
    OPTIONAL { ?franchise bb:franchiseName ?franchiseName . }
    OPTIONAL { ?franchise bb:active ?active . }

    ?team a bb:Team ;
          bb:franchiseOf ?franchise ;
          bb:yearID ?year .

    OPTIONAL { ?team bb:teamName ?teamName . }
    OPTIONAL { ?team bb:teamID ?teamId . }
    OPTIONAL { ?team bb:teamIDBR ?teamIdBr . }
    OPTIONAL { ?team bb:lgID ?league . }
    OPTIONAL { ?team bb:divID ?division . }
    OPTIONAL { ?team bb:Rank ?rank . }
    OPTIONAL { ?team bb:park ?park . }
    OPTIONAL { ?team bb:attendance ?attendance . }
    OPTIONAL { ?team bb:W ?wins . }
    OPTIONAL { ?team bb:L ?losses . }
}
ORDER BY ?franchiseId DESC(?year)
\end{Verbatim}
\end{itemize}

\subsection{Ligas}

\begin{itemize}[leftmargin=*]
  \item \textbf{/teams/league/\{league\_code\}/ -- Equipas da liga}
  
  Fun\c{c}\~ao: \texttt{get\_teams\_by\_league()}
  
  Descri\c{c}\~ao: devolve a lista de equipas/franquias da liga e a sua perman\^encia temporal.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>

SELECT ?teamName ?franchise (SAMPLE(?park) AS ?park)
       (COUNT(DISTINCT ?year) AS ?seasons)
       (MIN(?year) AS ?firstYear)
       (MAX(?year) AS ?lastYear)
WHERE {
    ?team a bb:Team ;
          bb:lgID ?lg ;
          bb:teamName ?teamName ;
          bb:yearID ?year .
    FILTER(?lg = "AL")
    OPTIONAL { ?team bb:franchID ?franchise . }
    OPTIONAL { ?team bb:park ?park . }
}
GROUP BY ?teamName ?franchise
ORDER BY ?teamName ?firstYear
\end{Verbatim}

  \item \textbf{/teams/league/\{league\_code\}/ -- S\'eries da liga}
  
  Fun\c{c}\~ao: \texttt{get\_league\_series\_results()}
  
  Descri\c{c}\~ao: devolve as séries em que a liga participou como vencedora ou vencida.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>

SELECT ?year ?round ?winnerTeamName ?loserTeamName ?wins ?losses ?ties ?lgWinner ?lgLoser
WHERE {
    ?series a bb:WorldSeriesResult ;
            bb:yearID ?year ;
            bb:round ?round .
    ?series bb:winnerTeam ?winnerTeam .
    ?winnerTeam bb:teamName ?winnerTeamName .
    ?series bb:loserTeam ?loserTeam .
    ?loserTeam bb:teamName ?loserTeamName .
    OPTIONAL { ?series bb:wins ?wins . }
    OPTIONAL { ?series bb:losses ?losses . }
    OPTIONAL { ?series bb:ties ?ties . }
    OPTIONAL { ?series bb:lgIDwinner ?lgWinner . }
    OPTIONAL { ?series bb:lgIDloser ?lgLoser . }
    FILTER(STR(?lgWinner) = "AL" || STR(?lgLoser) = "AL")
}
ORDER BY DESC(?year) ?round
\end{Verbatim}

  \item \textbf{/teams/league/\{league\_code\}/ -- L\'ideres estat\'isticos}
  
  Fun\c{c}\~ao: \texttt{get\_league\_leaders()}
  
  Descri\c{c}\~ao: devolve os melhores jogadores da liga para uma estatística e ano concretos.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?playerID ?nameFirst ?nameLast ?nameGiven ?name ?statValue
WHERE {
    ?stat a bb:BattingStat ;
          bb:lgID "AL" ;
          bb:yearID 2015 ;
          bb:homeRuns ?statValue .
    ?player bb:hasBatting ?stat ;
            bb:playerID ?playerID .
    OPTIONAL { ?player foaf:name ?name . }
    OPTIONAL { ?player bb:nameFirst ?nameFirst . }
    OPTIONAL { ?player bb:nameLast ?nameLast . }
    OPTIONAL { ?player bb:nameGiven ?nameGiven . }
}
ORDER BY DESC(?statValue)
LIMIT 3
\end{Verbatim}
\end{itemize}

\subsection{Grafo relacional}

\begin{itemize}[leftmargin=*]
  \item \textbf{/players/\{player\_id\}/ -- Grafo do detalhe do jogador}
  
  Fun\c{c}\~ao: \texttt{get\_player\_graph\_data(player\_id)}
  
  Descri\c{c}\~ao: constr\'oi o grafo relacional do jogador com equipas, franquias, ligas, pr\'emios, teammates e managers relacionados.
  
  Tipo SPARQL: \texttt{CONSTRUCT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

CONSTRUCT {
    ?player a bb:GraphPlayer ;
            bb:playerID ?playerCode ;
            foaf:name ?playerName .

    ?team a bb:GraphTeam ;
          bb:teamName ?teamName ;
          bb:teamID ?teamCode ;
          bb:yearID ?teamYear .

    ?franchise a bb:GraphFranchise ;
               bb:franchiseName ?franchiseName .

    ?leagueNode a bb:GraphLeague ;
                bb:lgID ?leagueCode .

    ?teammate a bb:GraphTeammate ;
              bb:playerID ?teammateCode ;
              foaf:name ?teammateName .

    ?managerPlayer a bb:GraphManager ;
                   bb:playerID ?managerCode ;
                   foaf:name ?managerName .

    ?award a bb:GraphAward ;
           bb:awardName ?awardName ;
           bb:yearID ?awardYear .

    ?player bb:playedFor ?team .
    ?team bb:franchiseLink ?franchise .
    ?team bb:leagueLink ?leagueNode .
    ?player bb:wonGraphAward ?award .
    ?player bb:sharedClubhouseWith ?teammate .
    ?team bb:managedByPerson ?managerPlayer .
}
WHERE {
    ?player a bb:Player ;
            bb:playerID ?playerCode .
    FILTER(?playerCode = "aaronha01")
  OPTIONAL { ?player foaf:name ?playerName . }
}
\end{Verbatim}
\end{itemize}

\subsection{Vistas auxiliares e rankings}

\begin{itemize}[leftmargin=*]
  \item \textbf{/analytics/ -- Líderes globais de jogadores}
  
  Fun\c{c}\~ao: \texttt{get\_global\_player\_leaders()}
  
  Descri\c{c}\~ao: devolve rankings globais agregados de jogadores para estatísticas acumuladas.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?playerID ?nameFirst ?nameLast ?nameGiven ?name (SUM(?statValue) AS ?totalStat)
WHERE {
    ?stat a bb:BattingStat ;
          bb:homeRuns ?statValue .
    ?player bb:hasBatting ?stat ;
            bb:playerID ?playerID .

    OPTIONAL { ?player foaf:name ?name . }
    OPTIONAL { ?player bb:nameFirst ?nameFirst . }
    OPTIONAL { ?player bb:nameLast ?nameLast . }
    OPTIONAL { ?player bb:nameGiven ?nameGiven . }
}
GROUP BY ?playerID ?nameFirst ?nameLast ?nameGiven ?name
ORDER BY DESC(?totalStat)
LIMIT 5
\end{Verbatim}
\end{itemize}

\subsection{Quiz interativo}

\begin{itemize}[leftmargin=*]
  \item \textbf{/quiz/api/start/ -- Banco de líderes estatísticos}
  
  Fun\c{c}\~ao: \texttt{get\_quiz\_leaderboard\_bank()}
  
  Descri\c{c}\~ao: recolhe líderes reais por liga e ano para construir perguntas de ranking.
  
  Tipo SPARQL: \texttt{SELECT}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?lg ?year ?playerID ?nameFirst ?nameLast ?nameGiven ?name ?statValue
WHERE {
    ?stat a bb:BattingStat ;
          bb:lgID ?lg ;
          bb:yearID ?year ;
          bb:homeRuns ?statValue .
    ?player bb:hasBatting ?stat ;
            bb:playerID ?playerID .
    FILTER(?year >= 2000)
    FILTER(?lg IN ("AL", "NL"))

    OPTIONAL { ?player foaf:name ?name . }
    OPTIONAL { ?player bb:nameFirst ?nameFirst . }
    OPTIONAL { ?player bb:nameLast ?nameLast . }
    OPTIONAL { ?player bb:nameGiven ?nameGiven . }
}
ORDER BY ?lg DESC(?year) DESC(?statValue) ?playerID
\end{Verbatim}

  \item \textbf{/quiz/api/answer/ -- Verifica\c{c}\~ao de líderes estatísticos}
  
  Fun\c{c}\~ao: \texttt{ask\_quiz\_leaderboard\_answer()}
  
  Descri\c{c}\~ao: confirma se a resposta escolhida é o verdadeiro líder dessa estatística.
  
  Tipo SPARQL: \texttt{ASK}

  SPARQL:

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>

ASK {
    ?player bb:playerID "arodial01" ;
            bb:hasBatting ?stat .
    ?stat a bb:BattingStat ;
          bb:lgID "AL" ;
          bb:yearID 2005 ;
          bb:homeRuns ?statValue .

    FILTER NOT EXISTS {
        ?otherStat a bb:BattingStat ;
                  bb:lgID "AL" ;
                  bb:yearID 2005 ;
                  bb:homeRuns ?otherValue .
        FILTER(?otherValue > ?statValue)
    }
}
\end{Verbatim}
\end{itemize}

\begin{Verbatim}[fontsize=\scriptsize,breaklines=true]
PREFIX bb: <http://baseball.ws.pt/>

PREFIX xsd: <http://www.w3.org/2001/XMLSchema#>

  ?subject a bb:Player ; bb:playerID "aaronha01" .
  OPTIONAL { ?subject bb:birthCountry ?currentValue . }
}
\end{Verbatim}
\end{itemize}
